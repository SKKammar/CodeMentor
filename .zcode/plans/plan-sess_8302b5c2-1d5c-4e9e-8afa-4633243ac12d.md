## Build Order (10 steps)

1. **Project scaffold** вЂ” `package.json`, `tsconfig.json`, `next.config.ts`, Tailwind/PostCSS config, `globals.css` with dark theme
2. **Supabase schema** вЂ” `supabase/schema.sql` with sessions + concept_graph tables + RLS policies
3. **Types & lib layer** вЂ” `types/index.ts`, `lib/supabase/client.ts`, `lib/supabase/server.ts`, `lib/claude.ts`, `lib/prompts.ts`
4. **Root layout + main page** вЂ” `app/layout.tsx` (Inter + JetBrains Mono fonts, dark theme), `app/page.tsx` (shell with header)
5. **Monaco Editor + UI components** вЂ” `MonacoEditor.tsx`, `ModeToggle.tsx`, `ProblemInput.tsx`, `AuthButton.tsx`, `SessionTracker.tsx`
6. **API routes** вЂ” `/api/chat` (streaming with 3 modes), `/api/session`, `/api/concepts`, `/api/dashboard`
7. **AI response panel** вЂ” `AiResponsePanel.tsx` with streaming markdown, syntax highlighting, `useChat` hook, `ConceptTags.tsx`
8. **Auth pages** вЂ” `/auth/login` with email+GitHub, `/auth/callback` route
9. **Dashboard** вЂ” `/dashboard/page.tsx` with stats cards, bar chart, line chart, streak tracker
10. **Knowledge graph** вЂ” `/dashboard/graph/page.tsx` with ReactFlow force-directed layout, glowing nodes, click-to-explain

## Key Implementation Details

- **Streaming**: `/api/chat` uses `ReadableStream` with `TextEncoder` to stream Claude tokens; frontend reads via `fetch` + `ReadableStream` reader
- **Socratic enforcement**: System prompt strictly forbids direct answers; conversation history maintained client-side and sent with each request
- **Dark theme**: GitHub dark palette (`#0d1117`, `#161b22`, `#30363d`, `#58a6ff`) via CSS custom properties + Tailwind
- **Framer Motion**: Fade-in on AI responses, slide-in on mode toggle, smooth transitions
- **RLS**: Supabase Row Level Security ensures users only see their own data
- **`.gitignore`**: node_modules, .env.local, .next/ all excluded
